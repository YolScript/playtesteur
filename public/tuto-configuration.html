⚙️ Configurer entièrement PlayTesteur

Ce guide couvre tous les réglages de la carte « Configuration du site » (page Compte, visible uniquement par le propriétaire). Chaque valeur enregistrée est appliquée immédiatement, sans redémarrage (sauf mention contraire) et remplace le fichier .env.

Vue d'ensemble — qui sert à quoi :
TuileSert àRéglages concernés
Connexion GoogleBouton « Se connecter avec Google »Client ID, Client Secret, URI de redirection
API Play ReviewsImport de fiche + validation automatique des avisClé JSON Play Console
API Google GroupsAjout automatique des testeurs aux groupes (optionnel)Clé JSON Groups, email admin, domaine

« DEV (simulé) » = fonctionnalité simulée en local, rien n'est cassé — juste pas encore branché sur le vrai service Google.

1. Connexion Google (OAuth) gratuit

  1. Ouvrez console.cloud.google.com et sélectionnez votre projet (ex : gen-lang-client-0484128331).
  2. Menu → API et services → Écran de consentement OAuth : type Externe, remplissez nom de l'app + email, enregistrez. Passez l'application en mode Production (sinon seuls les comptes de test peuvent se connecter).
  3. Menu → API et services → Identifiants → Créer des identifiants → ID client OAuth :
    • Type : Application Web
    • URI de redirection autorisée : https://VOTRE-SITE/api/auth/google/callback (remplacez par l'adresse réelle de votre site ; en local : http://localhost:8090/api/auth/google/callback)
  4. Copiez le Client ID et le Client Secret affichés.
  5. Dans la carte Configuration : collez-les dans GOOGLE_OAUTH_CLIENT_ID et GOOGLE_OAUTH_CLIENT_SECRET, et mettez la même URI de redirection dans GOOGLE_OAUTH_REDIRECT_URI. Enregistrez → la tuile passe en PRODUCTION.

2. API Play Console — avis & fiches gratuit

  1. Dans Google Cloud → IAM et administration → Comptes de service : utilisez le compte existant (playtesteur@gen-lang-client-0484128331.iam.gserviceaccount.com) ou créez-en un.
  2. Ouvrez le compte de service → onglet Clés → Ajouter une clé → Créer une clé → JSON. Un fichier .json se télécharge.
  3. Toujours dans Google Cloud → API et services → Bibliothèque : activez Google Play Android Developer API.
  4. Dans Play Console → Utilisateurs et autorisations → Inviter un nouvel utilisateur : collez l'email du compte de service, puis accordez :
    • Afficher les informations sur l'application (import de fiche)
    • Répondre aux avis (lecture des avis pour la validation des tests)
    • Gérer les canaux de test et modifier les listes de testeurs (recommandé)
  5. Ouvrez le fichier .json téléchargé avec le Bloc-notes, copiez tout son contenu, collez-le dans GOOGLE_PLAY_SERVICE_ACCOUNT_KEY_JSON. Enregistrez → tuile PRODUCTION.
⏳ L'invitation du compte de service dans Play Console peut mettre jusqu'à 24-48h à devenir pleinement active côté API.

3. Groupes de testeurs — deux options

Option A — Groupes gratuits @googlegroups.com recommandé, 0 €

Aucune configuration serveur nécessaire. La tuile « API Google Groups » restera sur DEV (simulé) : c'est normal et sans conséquence.

  1. Créez un groupe sur groups.google.com/creategroup avec votre compte Gmail habituel.
  2. Réglage « Qui peut rejoindre le groupe » → « Tout utilisateur du Web peut rejoindre » (adhésion en un clic) ou « …peut demander à rejoindre » (vous validez chaque testeur).
  3. Lors de la soumission de votre application sur PlayTesteur, collez l'adresse du groupe (xxx@googlegroups.com) dans le champ « Groupe Google de testeurs ».
  4. Dans Play Console → votre app → Tests → Test fermé → Testeurs : cochez Groupes et collez la même adresse.
  5. C'est tout : chaque testeur PlayTesteur reçoit automatiquement un bouton « Rejoindre le groupe Google » qui active son accès.

Option B — Gestion automatique par API (Workspace / Cloud Identity)

Nécessite un nom de domaine (~10 €/an). Cloud Identity Free est gratuit jusqu'à 50 utilisateurs ; Google Workspace Business Starter coûte ~6-7 €/mois.

  1. Achetez un domaine (OVH, Cloudflare, Namecheap...).
  2. Inscrivez-vous à Cloud Identity Free et vérifiez le domaine (enregistrement TXT à ajouter chez votre registrar).
  3. Dans admin.google.com → Sécurité → Contrôle des API → Délégation au niveau du domaine → Ajouter :
    • ID client : le numéro « Unique ID » du compte de service (visible dans Google Cloud → Comptes de service)
    • Champs d'application : https://www.googleapis.com/auth/admin.directory.group et https://www.googleapis.com/auth/admin.directory.group.member
  4. Dans la carte Configuration : remplissez GOOGLE_ADMIN_IMPERSONATE_EMAIL (votre compte admin du domaine, ex admin@votredomaine.fr), GOOGLE_GROUPS_DOMAIN (ex votredomaine.fr) et GOOGLE_SERVICE_ACCOUNT_KEY_JSON (même clé JSON qu'à l'étape 2, ou une autre).
  5. Enregistrez → tuile PRODUCTION : le site crée les groupes et ajoute/retire les testeurs tout seul.

4. Secret des sessions

SESSION_SECRET signe les cookies de connexion. Mettez une longue chaîne aléatoire (30+ caractères, générable sur n'importe quel générateur de mots de passe). Appliqué au prochain redémarrage du serveur ; tous les utilisateurs devront se reconnecter.

5. Vérifier que tout fonctionne

  • Les tuiles en haut de la carte passent de « DEV (simulé) » à « PRODUCTION » dès l'enregistrement (rechargez la page Compte).
  • Connexion Google : déconnectez-vous puis testez le bouton « Se connecter avec Google ».
  • Play Reviews : sur « Mes applications », essayez « Importer depuis Play Console » avec un vrai nom de package.
  • Groupes (option A) : rejoignez un test avec un second compte → le bouton « Rejoindre le groupe Google » doit s'ouvrir.
Sécurité : ces réglages ne sont visibles et modifiables que par le compte propriétaire (agorasjohn@gmail.com), vérifié côté serveur à chaque requête. Les clés et secrets enregistrés ne sont jamais renvoyés au navigateur.
