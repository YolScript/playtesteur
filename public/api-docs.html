PlayTesteurAPI

API de pilotage et de visualisation de l'éditeur vidéo/photo, conçue pour être utilisée par une IA (ou tout script d'automatisation navigateur).

L'éditeur PlayTesteur fonctionne 100% dans le navigateur — rien n'est envoyé au serveur. Il n'y a donc pas d'API HTTP : window.PlayTesteurAPI est un objet JavaScript exposé directement dans la page /#/editeur, piloté via CDP (Chrome DevTools Protocol) par des outils comme Playwright, Puppeteer, ou tout client compatible.

Se connecter

Depuis un script Playwright (ou équivalent) déjà attaché à la page de l'éditeur :

const etat = await page.evaluate(() => window.PlayTesteurAPI.getState());

const id = await page.evaluate(
  (url) => window.PlayTesteurAPI.addPhoto({ url, texte: 'Ma légende', duree: 3 }),
  'https://exemple.com/capture.png'
);

Toutes les méthodes qui chargent un média (addPhoto, setBackground, setMusic...) acceptent une URL (téléchargée via fetch puis chargée comme le ferait un <input type="file">) — data:, blob: et URLs same-origin fonctionnent toujours ; les URLs externes nécessitent que le serveur distant autorise CORS.

Chaque appel modifie le même état que l'interface utilisateur et rafraîchit le panneau à l'écran : un humain gardant l'onglet ouvert voit l'éditeur se mettre à jour en direct et peut reprendre la main à tout moment (undo/redo compris). Piloter par API n'est pas un mode exclusif.

État — getState()

Retourne un instantané JSON-sérialisable complet du projet (fond, musique, intro/outro, photos, blocs de texte, effets, format d'export, historique...). Les médias chargés ne sont jamais inclus directement (ce sont des objets DOM) — utilisez les champs hasMedia / hasLogo / hasImg / hasAudio pour savoir si un média est présent.

window.PlayTesteurAPI.getState()
// {

Export

Chaque fonction retourne le Blob produit — aucun téléchargement automatique n'est déclenché, l'appelant décide quoi en faire (sauvegarde locale via Playwright, envoi réseau, inspection...).

Mode IA — contours & capture complète

Pensé pour qu'une IA lise la disposition du projet (positions, tailles, chevauchements) sans avoir à interpréter une image détaillée, et pour repérer les calques placés hors du cadre d'export (invisibles dans la vidéo/image finale mais bien présents dans le projet).

setModeContours(actif) boolean

Remplace le rendu visuel de chaque calque (photo et texte) par un simple cadre pointillé + label (id, dimensions).

captureApercu() → dataURL

Capture une frame au cadrage normal (ce qui sera dans l'export), reflète le mode contours si actif.

captureVueComplete(zoomOut = 2.2) → dataURL

Capture avec un champ de vision élargi (recul de caméra), révélant les calques positionnés hors du cadre normal (x/y en dehors de 0..1). Combiner avec setModeContours(true) pour un vrai "plan du montage".

demarrerCaptureAuto(callback, intervalleMs = 2000) → fonction d'arrêt

Envoi automatique et périodique de captures (aperçu + vue complète + état) au callback fourni — utile pour un pilotage IA qui surveille la disposition en continu.

const arreter = window.PlayTesteurAPI.demarrerCaptureAuto(({ apercu, vueComplete, etat }) => {
  console.log('nb photos:', etat.photos.length);
}, 3000);
// arreter() pour stopper

Bonnes pratiques pour monter une vidéo complète

Leçons tirées d'un montage réel piloté par API (screenshots de site + narration). Chaque point corrige un piège rencontré concrètement — les suivre évite de refaire les mêmes erreurs.

1. Calculer le scale d'une photo depuis son ratio réel

Un scale fixe (ex. 0.3 pour tous les calques) laisse énormément de vide autour d'une image dont le ratio diffère du cadre 16:9 — en particulier une capture fullPage d'un site web, souvent bien plus haute que large. Calculez un scale qui remplit une fraction cible du cadre selon le ratio réel de l'image :

const RATIO_CADRE = 1920 / 1080;
function scaleAdapte(largeurImg, hauteurImg, margeCible = 0.6) {
  const ratioImg = largeurImg / hauteurImg;
  if (ratioImg > RATIO_CADRE) return margeCible; // image "large" : contrainte par la largeur
  return (1080 * margeCible * ratioImg) / 1920;  // image "haute" : contrainte par la hauteur
}

2. align ne déplace pas le bloc de texte, il aligne le texte à l'intérieur

Le panneau d'un bloc de texte reste toujours centré sur x, quel que soit align ('left'/'center'/'right'). Positionner x près d'un bord (ex. 0.94) fait déborder le panneau hors du cadre côté align:'right', puisque le panneau s'étend symétriquement autour de son centre. Pour une colonne latérale calée sur wrapWidth, calculez le centre en tenant compte de la largeur de la colonne :

const margeExterieure = 0.05;
const xGauche = margeExterieure + wrapWidth / 2;
const xDroite = 1 - margeExterieure - wrapWidth / 2;

3. Ne jamais coller un texte au bord ni le superposer à une photo

Gardez une marge extérieure d'au moins 5% (margeExterieure ci-dessus) et vérifiez la zone réellement occupée par la photo (dépend de son scale et de son ratio, voir point 1) avant de positionner un texte à proximité. Pour un montage "titre en haut + description latérale", décalez la photo d'un côté (ex. x: 0.6) pour libérer une vraie colonne de l'autre côté plutôt que de superposer texte et image.

4. Laisser une marge de silence textuel entre deux segments

Si la fenêtre [startTime, endTime] d'un texte touche exactement celle du suivant, les deux sont actives simultanément pendant l'animation d'entrée/sortie (~0.5s) — elles se superposent visuellement. Reculez le startTime et avancez le endTime d'environ 0.3s par rapport aux bornes du segment.

5. Préférer setTransition('none') pour un montage riche en texte

La transition 'fade' superpose brièvement les deux photos consécutives à l'écran — avec des mises en page différentes par segment (photo décalée à gauche/droite, texte différent), le résultat peut sembler confus. Une coupe franche ('none') reste lisible même avec beaucoup de contenu qui change à chaque segment.

6. Le titre intro/outro s'agrandit autour d'un centre fixe

Sa position verticale est ancrée à un centre fixe (indépendant de sa taille) : l'agrandir remonte son bord supérieur. Si un texte est positionné juste au-dessus (ex. description d'intro), remontez-le en conséquence pour ne pas le voir chevauché par le titre.

7. Vérifier le vrai format d'un logo avant de compter sur sa transparence

Un fichier .png peut en réalité être un JPEG (aucune transparence possible, quel que soit son contenu) — vérifiez avec un outil qui lit le format réel plutôt que l'extension avant de l'utiliser en logoUrl. Un logo à fond blanc opaque utilisé tel quel produira un carré blanc disgracieux dans l'intro/outro.

8. Caler la durée des segments sur la voix off, pas l'inverse

Si vous ajoutez une narration (voix off), générez d'abord les clips audio, mesurez leur durée réelle, puis fixez les durées de segment (durée du clip + une marge, ex. +1s) — plutôt que d'essayer de faire tenir une narration dans une fenêtre déjà choisie arbitrairement.

9. Un clic utilisateur réel doit toujours pouvoir (re)lancer l'audio

Un média chargé/joué par script (plutôt que par une vraie interaction) peut être bloqué par la politique autoplay du navigateur, et rester bloqué indéfiniment si rien ne retente la lecture ensuite. Si vous pilotez l'éditeur par script et que la musique reste silencieuse, un clic sur le bouton Play de l'aperçu relance explicitement tous les médias (musique, voix off, vidéo de fond) — c'est le point de rattrapage fiable, car un clic est toujours un vrai geste utilisateur.

10. page.evaluate n'a pas accès aux variables Node externes

Le callback passé à page.evaluate(fn, data) s'exécute dans le contexte du navigateur : seules les valeurs transmises explicitement via data (le second argument) sont accessibles à l'intérieur. Référencer une variable du scope Node englobant directement dans fn lève une erreur silencieuse ou une ReferenceError selon les cas.

Exemple complet

await page.evaluate(async () => {
  const API = window.PlayTesteurAPI;
  await API.setBackground({ type: 'media', url: '/Media/background.mp4' });
  await API.setMusic('/Media/musique.mp3', { fadeIn: 1, fadeOut: 2 });
  await API.setIntro({ active: true, texte: 'PLAYTESTEUR', duree: 3 });

  await API.addPhoto({
    url: '/shots/dashboard.png',
    texte: 'Suivez votre progression',
    duree: 3.2,
  });

  const id = await API.addTextBlock({
    texte: 'ÉDITEUR INTÉGRÉ',
    startTime: 5, endTime: 10,
    glowActive: true, glowColor: '#00e5ff',
  });

  await API.setOutro({ active: true, texte: 'REJOIGNEZ-NOUS', duree: 3 });
});

const blob = await page.evaluate(() => window.PlayTesteurAPI.exportMp4({ fps: 30 }));
